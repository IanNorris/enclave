### How OpenSpec is used in this repository

OpenSpec is used both as a vendored dependency and as the source of generated /
derived files. The following material in this repository originates from,
or is generated by, OpenSpec:

1. **`third_party/openspec/`** — the complete OpenSpec source, vendored verbatim
   as a git submodule pinned to v1.5.0. Its own `LICENSE` file is included there.

2. **`.github/skills/openspec-*/SKILL.md`** — agent skill files generated by
   `openspec init` (OpenSpec 1.5.0). Each file additionally self-identifies with
   `license: MIT` and `author: openspec` in its frontmatter.

3. **`.github/prompts/opsx-*.prompt.md`** — slash-command prompt files generated
   by `openspec init` (OpenSpec 1.5.0).

4. **`openspec/config.yaml`** and the `openspec/changes/` / `openspec/specs/`
   scaffold — the workspace layout and configuration created by `openspec init`.
   (The specific proposal / design / tasks / spec *content* authored for
   individual Enclave changes is original to Enclave, but it follows OpenSpec's
   structure and conventions.)

The Enclave web UI's **Specs** tab reads and reviews the `openspec/` directory,
and Enclave's agent tooling (`openspec_revision_log`) writes to OpenSpec's change
folders. These integrations are original Enclave code but exist to interoperate
with OpenSpec's format and workflow.
